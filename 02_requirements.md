# ポートフォリオ制作プロジェクト - 要件定義

## サイト構成

### ページ構成（2ページ）
1. **トップページ（Home）**
   - ヒーローセクション
   - 自己紹介（最小限）
   - 実績紹介（5件）
   - スキル・技術スタック
   - コンタクトへの導線

2. **コンタクトページ（Contact）**
   - 問い合わせフォーム
   - SNS・連絡先情報

---

## 機能要件

### 1. トップページ

#### ヒーローセクション
- **要素**:
  - 肩書き（例: Web Designer / Frontend Developer）
  - 提供価値を端的に示すサブヘッド
  - CTAボタン（コンタクトページへのリンク）
- **デザイン方針**:
  - クリーンでモダン
  - 抽象的な表現は避け、具体的な価値提案

#### 自己紹介セクション
- **要素**:
  - 簡潔な自己紹介文（2-3文程度）
  - 提供できる価値・強み
  - プロフィール写真（オプション）
- **注意点**:
  - 自分語りは最小限に
  - クライアントのメリットにフォーカス

#### 実績紹介セクション（Works）
- **掲載数**: 5件（厳選）
- **各実績に含める情報**:
  1. プロジェクト名
  2. サムネイル画像
  3. 課題（Challenge）: クライアントが抱えていた課題
  4. 解決策（Solution）: どのようにアプローチしたか
  5. 結果（Result）: 定量的・定性的な成果
  6. 使用技術・ツール
  7. プロジェクト期間
  8. 担当範囲（デザイン/実装/両方）

- **表示形式**:
  - カード型レイアウト
  - ホバーエフェクトで詳細表示
  - モーダルまたは専用ページで詳細閲覧（検討）

#### スキル・技術スタックセクション
- **カテゴリ**:
  1. デザインツール
     - Figma
     - Adobe XD / Photoshop / Illustrator（使用経験がある場合）

  2. フロントエンド開発
     - HTML / CSS / JavaScript
     - React / Next.js
     - TypeScript
     - TailwindCSS

  3. バックエンド・インフラ
     - Node.js（基本レベル）
     - Vercel
     - GitHub

  4. その他ツール
     - Shopify / Shopify Liquid
     - Studio（ノーコードツール）
     - Git / GitHub
     - ESLint / Prettier

- **表示形式**:
  - アイコン + テキスト
  - レベル表示は行わず、使用可能な技術として列挙

### 2. コンタクトページ

#### 問い合わせフォーム
- **フォーム項目**:
  1. 名前（必須）
  2. メールアドレス（必須）
  3. 件名（必須）
  4. お問い合わせ内容（必須・テキストエリア）

- **機能要件**:
  - バリデーション実装
    - 必須項目チェック
    - メールアドレス形式チェック
    - 文字数制限
  - 送信確認モーダル
  - 送信完了メッセージ
  - エラーハンドリング

- **技術実装**:
  - 自作コンタクトフォーム（外部サービス非依存）
  - SendGrid等のメールAPIを使用
  - Next.js API Routesで実装

#### 連絡先情報
- メールアドレス（クリッカブル）
- SNSリンク（GitHub、Twitter/X、LinkedIn等）
- 営業時間・返信目安時間（オプション）

## 非機能要件

### パフォーマンス
- **目標指標**:
  - Lighthouse Score: 90点以上（全項目）
  - First Contentful Paint (FCP): 1.8秒以下
  - Time to Interactive (TTI): 3.8秒以下

- **最適化手法**:
  - 画像の最適化（WebP形式、遅延読み込み）
  - コード分割
  - CDN活用（Vercel）

### レスポンシブ対応
- **対応デバイス**:
  - デスクトップ（1920px, 1440px, 1280px）
  - タブレット（768px, 1024px）
  - モバイル（375px, 414px）

- **アプローチ**:
  - モバイルファースト設計
  - TailwindCSSのブレークポイント活用

### ブラウザ対応
- **対応ブラウザ**:
  - Chrome（最新版）
  - Firefox（最新版）
  - Safari（最新版）
  - Edge（最新版）

### アクセシビリティ
- **基準**: WCAG 2.1 AA レベル準拠を目指す
- **実装項目**:
  - セマンティックHTML
  - 適切なARIA属性
  - キーボード操作対応
  - 十分なコントラスト比

### SEO
- **基本設定**:
  - メタタグ設定（title, description, OGP）
  - 構造化データ（JSON-LD）
  - sitemap.xml生成
  - robots.txt設定

## コンテンツ要件

### 掲載実績の選定基準
1. **多様性**: 異なる業種・課題を示せる案件
2. **成果の明確性**: 定量的または定性的な結果が示せるもの
3. **技術的な見せ場**: 使用した技術スタックが明確なもの
4. **守秘義務**: 公開可能な案件のみ

### テキストトーン
- プロフェッショナルかつ親しみやすい
- 専門用語は使うが、必要に応じて補足
- 「です・ます」調で統一
- 具体的で定量的な表現を心がける

---

  - Button
  - Card
  - Input / Textarea
  - Modal
  - Navigation
  - Footer

- **設計方針**:
  - Atomic Design的なアプローチ
  - props経由でのカスタマイズ性
  - アクセシビリティを考慮した実装

## リスク管理

### 想定リスクと対策

1. **技術的な障壁**
   - リスク: Next.js App Routerの学習コスト
   - 対策: 公式ドキュメント参照、シンプルな構成から開始

2. **コンテンツ不足**
   - リスク: 公開可能な実績が5件に満たない
   - 対策: 架空のケーススタディ、個人プロジェクトの追加

3. **スケジュール遅延**
   - リスク: デザイン・実装に想定以上の時間
   - 対策: MVP（最小限の機能）を優先、段階的リリース

4. **メール送信の不具合**
   - リスク: フォーム送信エラー、スパム判定
   - 対策: 送信テストの徹底、SendGrid等の信頼性の高いサービス使用
