# ポートフォリオ制作プロジェクト - 戦略・コンセプト

## プロジェクト概要

個人Webデザイナーとしてのポートフォリオサイト制作プロジェクト。
「Goodpatchを意識した高い技術力」と「個人事業主への親しみやすさ」を両立させることを目指す。

**公開URL**: `design.mutalog.com`（サブドメイン運用）

---

## 目的とターゲット

### 主目的
**営業・集客 + 就職活動に対応**した汎用性の高いポートフォリオサイト

#### 目的1: 営業・集客ツール（優先度: 高）
- 個人事業主や小規模事業者からの案件獲得
- 名刺のQRコード、SNSリンクからの流入を想定
- 営業資料（PDF）を補完する役割
- デザインとコーディングの両面での技術力の証明

#### 目的2: 就職活動ツール（グッドパッチ等）
- デザイン企業への応募時の補助資料
- 就活用PDF資料を補完する役割
- モダンな技術スタックによる実装力の証明
- プロセス思考・問題解決力の可視化

**重要**: どちらの目的でも、**Webサイトは副産物・補完的な役割**。メインは営業資料・就活用PDF。

### ターゲット

#### ターゲット1: 営業（クライアント）
- 個人事業主・小規模事業者
- 「解決策」を求めているクライアント
- デザインだけでなく実装力も重視する層
- WebサイトやECサイトのリニューアルを検討している事業者

#### ターゲット2: 就活（採用担当者・デザイナー）
- Goodpatch等のデザイン企業の採用担当者
- モダンな技術スタックを重視するエンジニア・デザイナー
- プロセスや思考過程を評価する企業文化

---

## コアコンセプト

### 1. プロフェッショナルな技術力の証明
- Goodpatchのエンジニア採用要件をトレース
- モダンな技術スタックの採用により、技術的信頼を担保
- 「コードで実装する力」を明示的に示す

### 2. 実績の質的厳選
- 表面的なデザインだけでなく「課題・解決・結果」を明記
- 5件に絞った厳選実績により、認知負荷を抑える
- プロセスの言語化によって思考力を可視化

### 3. ミニマムで洗練された情報設計
- 抽象的なキャッチコピーを排除
- 実績と提供価値をダイレクトに伝える
- 2ページ構成（トップ + コンタクト）のシンプルさ

## 差別化ポイント

### やること（積極的に打ち出す）
1. **技術スタックの明示**
   - React / Next.js / TypeScript / TailwindCSS
   - ESLint / Prettier による品質管理

2. **プロセスの可視化**
   - 課題定義 → 解決策 → 結果 の流れを明確に記述
   - デザインシステム・コンポーネント指向の採用

3. **自作実装へのこだわり**
   - コンタクトフォームを外部サービスに頼らず自作
   - GitHub + Vercel によるモダンな開発環境

### やらないこと（あえて削ぎ落とす）
1. **ポエティックな表現**
   - 「デザインでつくるカタチ」のようなキャッチコピーは不使用
   - 提供価値を直接的に伝える言語選択

2. **大量の作品掲載**
   - 質を重視し、5件に絞る

3. **長い自己紹介**
   - 自分語りは最小限に
   - クライアントが求める「解決策」にフォーカス

4. **過度に詳細な情報**
   - 職務経歴の詳細はPDF版（営業資料・就活資料）に集約
   - Webサイトは「入口・概要」として機能させる

5. **ノーコードツールでの構築**
   - このサイト自体で「コードによる実装力」を示すため、ノーコードは使わない
   - （ただし実績としてStudio/Shopifyでの制作物は掲載）

## ブランディング方針

### トーン＆マナー
- プロフェッショナルかつフレンドリー
- 技術的な正確さと、わかりやすさの両立
- 実績ベースの説得力

### ビジュアル方向性
- クリーンで洗練されたデザイン
- コンポーネント指向を活かしたシステマティックなUI
- Goodpatch的なモダンさと、個人事業主が親しみを感じる温度感のバランス

---

## 成功指標（KPI想定）

### 営業・集客における成功指標

#### 定性的指標
- 問い合わせ時のクライアントの質（プロジェクトの質・予算感）
- 技術力への言及の有無
- 信頼感・安心感のフィードバック

#### 定量的指標
- 月間問い合わせ件数
- サイト滞在時間
- 実績ページの閲覧率
- QRコード・SNS経由のアクセス数

### 就活における成功指標

#### 定性的指標
- 書類選考通過率
- 面接時の「ポートフォリオサイトを見た」という言及
- 技術スタックへの評価・質問

#### 定量的指標
- 採用担当者からのアクセス（LinkedIn経由等）
- GitHubリポジトリへのアクセス数
- サイト全体の滞在時間（詳細まで見られているか）
